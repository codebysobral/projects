# 4 Differences Between Git Rebase and Git Merge

| **Git Rebase** | **Git Merge** |
|----------------|---------------|
| Rewrites history | Preserves history |
| Linear history | Branch history |
| No merge commits | Creates merge commits |
| Best before sharing | Best for shared branches |


## When would I use Git Rebase?

- Clean up commit history.
- Update a feature branch.
- Before opening a pull request.
